TeamFlow AI — 从零搭建 Spring Boot + LangChain4j AI Agent 平台

发表于2026-05-26|更新于2026-05-26|后端开发
|浏览量:

项目背景

作为实验室负责人，我需要一个团队内部的 AI 工具来提升科研效率——代码审查、知识问答、文献分析。市面上的方案大多是 Python 生态（LangChain），但团队技术栈是 Java。于是基于 Spring Boot 3.2 + LangChain4j 1.0 从零搭建了这个平台，130+ 源文件独立开发，已部署上线。

项目源码：github.com/jasmine5606/studio-ai
在线演示：studio-ai.up.railway.app

核心架构

一、Agent 工具调用 + MCP 协议

使用 LangChain4j 的 @Tool 注解让 AI 模型能自主调用 GitHub API（仓库文件读取、PR 差异获取等）。同时引入 MCP（Model Context Protocol） 协议，通过 stdio 传输层动态扩展外部工具，理论可扩展至 89 种 GitHub 操作。

MCP 类似于 LSP（Language Server Protocol）之于 IDE——它为 AI Agent 提供了标准化的工具通信协议，解决了不同工具源的接入成本问题。

二、混合检索 RAG（Pinecone + Redis BM25）

纯向量检索在专业术语匹配上存在盲区——“Redis 持久化策略”这类精确查询，向量相似度可能不如关键词匹配。于是设计了一套双路召回架构：

向量检索（Pinecone）

  • 使用 DashScope 的 text-embedding-v3 模型（1024 维）
  • 语义匹配，适合模糊查询和长文本理解

BM25 倒排索引（Redis）

  • 自研中文 Bigram 分词器
  • 基于 Redis Sorted Set 实现，支持增量写入和快照持久化
  • 精确匹配，适合代码片段、专业术语

加权融合

Min-Max 归一化后加权融合，召回率从纯向量的 72% 提升至 91%。

三、Prompt 工程与 Code Review

迭代了三版 Prompt 模板，误报率从 40% 降至 15%：

版本 策略 误报率
V1 简单描述 40%
V2 添加”仅报告逻辑错误/安全漏洞”约束 25%
V3 引入 Few-shot 示例 15%

同时建立了反馈闭环——用户可对审查结果进行”误报/有效”标记，按规则类型分组统计，指导下一轮 Prompt 迭代方向。

四、多模态知识接入

打通了完整的音视频知识入库链路：

基于 Apache Tika 支持 txt / md / pdf / docx / ppt / mp3 / m4a 等 7 种格式的统一处理。

五、分层记忆系统

短期记忆

  • 滑动窗口：最近 10 条对话作为上下文注入
  • 摘要压缩：超过 30 条时触发 LLM 生成摘要
  • Redis 持久化：24h TTL 自动过期
  • 容错降级：Redis 不可用时自动切 ConcurrentHashMap 内存存储

长期记忆

  • 用户主动 Pin 关键信息
  • 向量嵌入 → Pinecone 存储
  • 元数据过滤（memoryType=pin + userId）实现用户隔离召回

六、工程化与部署

  • SSE 流式响应：逐 token 推送 AI 回答，体验接近 ChatGPT
  • WebSocket 协作：房间模型多人同时评审
  • Bucket4j 限流：令牌桶算法，默认 60 req/min
  • 认证：自研 Bearer Token 会话认证 + JWT 无状态认证双轨
  • 审计日志：@Async 异步记录，不阻塞主流程
  • 数据脱敏：手机号/邮箱/身份证/API Key 自动检测与掩码
  • 定时备份：每日凌晨 3 点自动备份关键数据，30 天保留

技术栈总结

层级 技术选型
框架 Spring Boot 3.2.5, Spring Data JPA
AI 框架 LangChain4j 1.0.0-beta3
LLM DashScope Qwen-Max (OpenAI 兼容 API)
向量库 Pinecone (生产) / InMemory (开发)
数据库 PostgreSQL (生产) / H2 (开发) / Redis
解析 Apache Tika 2.9.2
运维 Docker Compose, Railway, Prometheus, Actuator
前端 React 18 + TypeScript + Tailwind CSS + Zustand

总结

LangChain4j 是 Java 生态中 AI Agent 开发的首选框架，声明式的 @AiService + Spring DI 让开发体验非常流畅。混合检索是 RAG 落地的关键——纯向量不够，BM25 补充精确匹配能力。MCP 协议让 Agent 的工具链具备标准化扩展能力，值得持续关注。
