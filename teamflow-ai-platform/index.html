TeamFlow AI — 从零搭建 Spring Boot + LangChain4j AI Agent 平台

发表于2026-05-26|更新于2026-05-26|项目实战
|浏览量:

项目背景

作为实验室负责人，需要一个团队内部的 AI 工具来提升科研效率——代码审查、知识问答、文献分析。市面上方案大多 Python 生态，团队技术栈是 Java。基于 Spring Boot 3.2 + LangChain4j 1.0 从零搭建。

核心架构

技术亮点

Agent 工具调用 + MCP 协议

使用 LangChain4j @Tool 注解让 AI 能自主调用 GitHub API。引入 MCP（Model Context Protocol） 协议，通过 stdio 传输层动态扩展外部工具。

混合检索 RAG（Pinecone + Redis BM25）

纯向量检索在专业术语匹配上存在盲区，设计双路召回架构：

  • Pinecone 向量检索：语义匹配，适合模糊查询
  • Redis BM25 倒排索引：精确匹配，适合代码片段、专业术语
  • 加权融合：Min-Max 归一化后 0.6:0.4 加权，召回率从 72% 提升至 91%

Prompt 工程与 Code Review

迭代了三版 Prompt 模板，误报率从 40% 降至 15%：

  • V1：简单描述 → 过度敏感
  • V2：添加”仅报告逻辑错误/安全漏洞”约束 → 误报率 25%
  • V3：引入 Few-shot 示例 → 误报率 15%

分层记忆系统

  • 短期记忆：滑动窗口（10 条）+ Redis 持久化 + LLM 摘要压缩（30 条触发）
  • 长期记忆：用户 Pin → 向量嵌入 → Pinecone 存储 → 元数据过滤隔离

技术栈

层级 技术
框架 Spring Boot 3.2.5
AI LangChain4j 1.0.0-beta3, DashScope Qwen-Max
数据库 PostgreSQL、Redis
向量库 Pinecone
前端 React 18 + TypeScript + Tailwind CSS
部署 Docker Compose, Railway
